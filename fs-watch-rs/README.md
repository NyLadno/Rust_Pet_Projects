# fs-watch-rs

Кроссплатформенная CLI-утилита для мониторинга файловой системы в реальном времени.

Наблюдает за указанной директорией, фиксирует все изменения (создание, изменение, удаление, переименование файлов), фильтрует события по заданным критериям и доставляет уведомления через настраиваемые каналы вывода (stdout, файл, вебхук).

## Особенности
* 🚀 **Производительно**: Написано на Rust (edition 2024), минимальное потребление CPU и памяти
* 🔄 **Множественные репортеры**: Вывод в консоль с цветным оформлением, запись в лог-файл, отправка HTTP POST Webhook (асинхронно)
* 🎯 **Гибкая фильтрация**: Фильтрация событий по расширению (`.rs`, `.toml`) или glob-паттерну (`*.log`)
* ⚙️ **Единый конфиг**: Поддержка настроек через аргументы CLI и файл конфигурации `config.toml`
* ⏱️ **Дебаунсинг**: Группировка повторяющихся событий (решает проблему множественных "Modify" при сохранении файла IDE)

## Установка

Для сборки и установки из исходников необходим [Rust (cargo)](https://rustup.rs/):

```bash
# Клонирование репозитория
git clone https://github.com/yourusername/fs-watch-rs.git
cd fs-watch-rs

# Сборка и установка бинарника (появится в ~/.cargo/bin)
cargo install --path .
```

### Использование Docker

В репозитории есть готовый `Dockerfile` для создания минималистичного образа:

```bash
# Сборка Docker-образа
docker build -t fs-watch-rs .

# Запуск контейнера с пробросом наблюдаемой папки
# ВАЖНО: Папка для мониторинга монтируется через volume (например, -v $(pwd):/workspace)
docker run --rm -v $(pwd):/workspace fs-watch-rs /workspace -r
```

## Использование

### Базовый мониторинг
Мониторинг текущей директории с цветным выводом в терминал:
```bash
fs-watch-rs .
```

### Наблюдение с фильтрами и логом
Рекурсивный мониторинг файлов `.rs` и `.toml`, с записью событий в `events.log`:
```bash
fs-watch-rs ./src --recursive --filter .rs,.toml --output ./events.log
```

### Использование Webhook
Отправка POST-запросов (JSON) на указанный URL при изменениях (дебаунс 500мс):
```bash
fs-watch-rs ./configs -r --webhook https://hooks.example.com/notify --debounce 500
```

### Работа с конфиг-файлом
Приложение умеет читать настройки из `config.toml`. CLI-аргументы имеют приоритет над файлом конфигурации.

Запуск с указанием пути к конфигурации:
```bash
fs-watch-rs --config ./config.toml
```

Пример `config.toml`:
```toml
watch_path  = "./src"
recursive   = true
debounce_ms = 300
filters     = [".rs", ".toml"]

[output]
file        = "./watch.log"

[webhook]
url         = "https://hooks.example.com/notify"
timeout_sec = 5
```

## JSON Webhook Payload
При срабатывании вебхука отправляется следующий payload:
```json
{
  "event": "Modified",
  "path": "/home/user/project/src/main.rs",
  "timestamp": "2026-04-13T14:32:45+03:00",
  "extension": "rs"
}
```
